Mas além de coisas relacionadas ao baixo nível, eu também aprendi linguagens como:

  • Bash

Embora já tenha feito alguns trabalhos freelances, eu ainda não considero que tenho experiência na área. Mas estou empenhado em estudar para tentar entrar na área de desenvolvimento WEB, que acredito que será um excelente começo na minha carreira.

Ainda não é a área que almejo, mas aprender é meu principal interesse. E aprender na prática como sistemas WEB são desenvolvidos não é uma má ideia. 🙂

Atualmente o projeto mais legal que eu tenho no meu GitHub é o Lia, uma linguagem de programação com sintaxe no estilo de Assembly.

O compilador do Lia eu escrevi em C e fiz testes unitários usando outro projeto meu, o metric. Se estiver curioso, pode ler sobre o projeto clicando aqui.

  • SOSE — Um emulador WEB para o microprocessador 8086 escrito em JavaScript.
  • SliB.js — Biblioteca JavaScript para o desenvolvimento de jogos 2D.
  • Ases — Interpretador em C de uma linguagem de programação esotérica que eu mesmo criei.
  • metric — Kit de macros na linguagem C para escrita de testes unitários e benchmark de trechos do programa.
  • cli-tools — Algumas ferramentas em Bash, incluindo ferramenta para template de arquivos de texto e binários.
  • asm-tools — Ferramentas escritas em Assembly para servirem como código de exemplo.
  • livro-assembly-x86 — Livro gratuito sobre Assembly da arquitetura x86 e x86-64.

cat Projetos/livro-assembly-x86/README.md

Gosto bastante de espalhar conhecimento, e desde o momento que tive oportunidade de acessar a internet é o que venho fazendo. Inicialmente ajudando em fóruns sobre desenvolvimento de jogos, só que mais recentemente eu ajudo em grupos no Facebook. Também já escrevi alguns artigos no Medium e fiz alguns vídeos no YouTube.

Atualmente estou escrevendo um livro sobre Assembly da arquitetura x86 e x86-64, afim de partilhar o que aprendi. Quando eu comecei a aprender Assembly o material em português era escasso e de péssima qualidade. Até hoje se encontra alguns cursos no YouTube, mas todos que eu vi são muito superficiais e não ensinam nada.

Devido a esta escassez na nossa língua, eu resolvi escrever este livro que tenta ser uma introdução fácil porém profunda para a linguagem. Evito ao máximo ser superficial e gosto bastante que as pessoas realmente entendam o que estão fazendo, e não somente decorem como fazer.

Minha principal fonte para escrever este livro são os próprios manuais para desenvolvedores da Intel. Só que este material é muito pesado para um iniciante, são 5038 páginas de puro conteúdo técnico. Além de ser escrito na língua inglesa, o que pode ser um problema para muitos aqui no Brasil.

Você pode ler o livro gratuitamente no link abaixo:

  • https://silva97.gitbook.io/assembly-x86/
